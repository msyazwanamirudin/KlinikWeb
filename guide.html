11. Security Features

Login Lockout

After 3 failed login attempts, the system locks out to prevent brute-force attacks. To recover:

  • Click the "Forgot access?" link below the login form
  • This clears the lockout and lets you try again

Owner Password Protection

Dangerous actions require the owner password before proceeding:

  • Deleting inventory items
  • Deleting roster rules
  • Deleting promo items
  • Accessing the Admin tab

Developer Tools Disabled

To prevent casual source code inspection, the following are blocked on the admin page:

  • Right-click context menu
  • F12, Ctrl+Shift+I, Ctrl+Shift+J, Ctrl+Shift+C, Ctrl+U

Password Hashing

All passwords are hashed using HMAC-SHA256 with a salt before comparison. Passwords are never stored or transmitted in plain text.

12. Firebase Usage & Tips

The system uses Firebase Realtime Database on the free Spark plan:

Resource Free Limit How to Minimize
Storage 1 GB Upload compressed images, trim old expenses
Monthly Bandwidth 10 GB Minimize tab switching, close inactive sessions
Connections 100 simultaneous Limit admin sessions to 1-2 at a time

Automatic Optimizations

  • ‚úÖ Promo images are auto-compressed (JPEG, 800px, 60% quality)
  • ‚úÖ Cash flow entries older than 12 months are auto-trimmed
  • ‚úÖ Bandwidth usage data older than 3 months is auto-cleaned
  • ‚úÖ Tab data is cached locally to reduce redundant database reads

Best Practices

  1. Use external image URLs for promo instead of uploading when possible
  2. Don't leave the admin page open all day ‚Äî each listener uses bandwidth
  3. Check the Analytics tab ‚Üí Firebase Usage section regularly
  4. Keep promo count to 3-5 items maximum
  5. Export cash flow data monthly for your own records

13. Troubleshooting

Problem Solution
Can't log in ‚Äî "System locked" Click "Forgot access?" to clear the lockout, then try again with correct credentials.
Admin tab not visible You must be logged in as the owner (owner@klinik.com). The Admin tab is hidden for admin accounts.
Owner Password prompt won't accept my password The owner password is separate from your login password. Default is 1234.
Image upload shows "Compressing..." forever The image may be corrupted. Try a different file. Only JPEG, PNG, WebP, and GIF are supported (max 500KB).
Data not updating on the website Hard refresh the public website (Ctrl+Shift+R). Firebase updates may take a few seconds to propagate.
Firebase usage showing high Check the Analytics tab for details. Consider removing unused promo images (they're stored as base64 in the database).
Expenses from last year disappeared Entries older than 12 months are automatically trimmed. Export data monthly to keep records.
